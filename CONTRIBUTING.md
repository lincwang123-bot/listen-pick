## Development

```bash
npm install
npm start
npm test
```

## Content Guidelines

All learning content must be suitable for Chinese children ages 6-10.

Use sentences that can be illustrated in five seconds. Avoid abstract ideas, adult work scenes, unsafe behavior, frightening images, culturally narrow references, and exam-style trick questions.

### Illustration Quality Gate

All production lesson images must match the finished quality of the existing Level 1-100 picture-card illustrations. New levels must use warm, polished children's picture-book illustrations with cohesive character design, clean outlines, clear full-body or clearly framed subjects, soft lighting, and low-distraction backgrounds.

Do not ship simplified vector placeholders, stick-figure drawings, geometric icon scenes, rough SVG mockups, cropped limbs, distorted hands, text, labels, numbers, captions, logos, or watermarks as final learning assets. Programmatic drawings are allowed only as temporary semantic placeholders for internal review and must be replaced before release or public testing.

## Pull Request Checklist

- The app still runs locally.
- `npm test` passes.
- New English sentences are grammatically correct.
- Chinese meanings are natural and age-appropriate.
- Correct and wrong images differ by one clear learning point.
- New production illustrations match the Level 1-100 quality bar and are not placeholder/vector mockups.
- No secrets, local paths, browser profiles, or temporary generated folders are committed.
